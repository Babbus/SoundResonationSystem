  <action>
    Create `ResonanceAudioBridge.cs` in `Assets/SoundResonance/Runtime/Hybrid/` as a MonoBehaviour in namespace `SoundResonance`:

    **Constants:**
    - `public const int MaxVoices = 16;`

    **Fields:**
    - `private NativeArray<VoiceData> _bufferA;` and `_bufferB;` -- the double-buffer, each length MaxVoices
    - `private volatile int _readIndex;` -- 0 = read A, 1 = read B (audio thread reads whichever _readIndex points to)
    - `private VoicePool _voicePool;`
    - `private HashSet<int> _previouslyActiveEntityIds;` -- track which entities were active last frame for IsNewStrike detection
    - `private HashSet<int> _currentlyActiveEntityIds;` -- this frame's active entities

    **Public API (called by VoiceSynthesizer on audio thread):**
    - `public NativeArray<VoiceData> GetReadBuffer()` -- returns `_readIndex == 0 ? _bufferA : _bufferB;`
    - `public int ReadIndex => _readIndex;` -- for synthesizers to know which buffer to read

    **Lifecycle:**
    - `void Awake()`:
      - Allocate _bufferA and _bufferB as `new NativeArray<VoiceData>(MaxVoices, Allocator.Persistent)`
      - Create `_voicePool = new VoicePool(MaxVoices);`
      - Initialize _previouslyActiveEntityIds and _currentlyActiveEntityIds as empty HashSets
    - `void OnDestroy()`:
      - Dispose _bufferA and _bufferB (check IsCreated first)

    **LateUpdate (main thread, runs after all ECS systems):**
    1. Get EntityManager: `var world = World.DefaultGameObjectInjectionWorld; if (world == null || !world.IsCreated) return; var em = world.EntityManager;`
    2. Build query for active emitters: `em.CreateEntityQuery(ComponentType.ReadOnly<EmitterTag>(), ComponentType.ReadOnly<ResonantObjectData>(), ComponentType.ReadOnly<LocalTransform>())` -- default query filtering only returns entities where EmitterTag is ENABLED (active emitters)
    3. Get arrays: `query.ToEntityArray(Allocator.Temp)`, `query.ToComponentDataArray<ResonantObjectData>(Allocator.Temp)`, `query.ToComponentDataArray<LocalTransform>(Allocator.Temp)`
    4. Determine write buffer: `var writeBuffer = _readIndex == 0 ? _bufferB : _bufferA;`
    5. Clear all slots in writeBuffer (set Active = 0)
    6. Swap _currentlyActiveEntityIds to previous, clear current
    7. For each active entity (up to MaxVoices):
       - `int entityId = entities[i].Index;`
       - `_currentlyActiveEntityIds.Add(entityId);`
       - `int slot = _voicePool.GetOrAssign(entityId, dataArray[i].CurrentAmplitude);`
       - `_voicePool.UpdateAmplitude(slot, dataArray[i].CurrentAmplitude);`
       - Determine `isNewStrike`: entity is in current but was NOT in _previouslyActiveEntityIds
       - Determine `isSympathetic`: check if entity had a StrikeEvent enabled this frame. Actually, since EmitterActivationSystem already consumed the StrikeEvent by LateUpdate, we cannot check StrikeEvent directly. Instead: if `isNewStrike` is true AND `EmitterTag.StrikeAmplitude > 0`, it was a direct strike. If `isNewStrike` is true AND the entity appeared without a strike (sympathetic activation), mark as sympathetic. SIMPLER approach: use the EmitterTag.StrikeAmplitude field. If StrikeAmplitude > 0 on a newly appearing entity, it was directly struck. Read EmitterTag component: `em.GetComponentData<EmitterTag>(entities[i])`. If isNewStrike and emitterTag.StrikeAmplitude > 0, it's a direct strike (IsNewStrike=1, IsSympathetic=0). If isNewStrike and StrikeAmplitude == 0 (or very small), it's sympathetic (IsNewStrike=0, IsSympathetic=1).
       - Populate VoiceData struct and write to writeBuffer[slot]
    8. Release voices for entities that were in _previouslyActiveEntityIds but not in _currentlyActiveEntityIds
    9. Atomic swap: `_readIndex = _readIndex == 0 ? 1 : 0;`
    10. Dispose temp arrays

    **Important threading note:** `volatile int _readIndex` provides the happens-before guarantee. Main thread completes all writes to the write buffer BEFORE flipping _readIndex. Audio thread reads _readIndex first, then reads the buffer -- the volatile ensures it sees the completed writes.

    **Do NOT use locks** -- the double-buffer pattern is lock-free by design. The audio thread reads one buffer while main thread writes the other. The only shared state is _readIndex (a single int, atomically readable).

    Import requirements: `using Unity.Entities;`, `using Unity.Transforms;`, `using Unity.Collections;`, `using Unity.Mathematics;`, `using System.Collections.Generic;`
