  <name>Task 1: Create StrikeInputManager MonoBehaviour</name>
  <files>
    Assets/SoundResonance/Runtime/Input/StrikeInputManager.cs
  </files>
  <action>
    Create the Input/ directory under Runtime/ if it does not exist. Create StrikeInputManager.cs -- a MonoBehaviour that bridges mouse clicks to ECS StrikeEvent components.

    **Architecture:** No-SubScene approach. Resonant objects live directly in the scene (not in a SubScene), so they retain standard BoxCollider components. This means Physics.Raycast is the primary and only pick method -- simple and reliable.

    **Input detection:** On each Update, check `Mouse.current.leftButton.wasPressedThisFrame`. If true, proceed to raycasting.

    **Physics.Raycast picking:**
    1. Get mouse screen position from `Mouse.current.position.ReadValue()`
    2. Create a ray via `Camera.main.ScreenPointToRay(mousePos)`
    3. Perform `Physics.Raycast(ray, out RaycastHit hit, maxDistance)` with configurable `maxDistance` (default 100f)
    4. If hit, check if the hit object has a `ResonantObjectAuthoring` component via `hit.collider.GetComponent<ResonantObjectAuthoring>()`
    5. If found, get the entity from the authoring component. Use the authoring component to look up the baked entity -- store the entity reference on the authoring component or use `GetEntity()` approach from the Baker. The simplest approach: add a `public Entity BakedEntity;` field to ResonantObjectAuthoring that the Baker sets, OR query EntityManager to find the entity matching this GameObject.
    6. Enable StrikeEvent on that entity: `entityManager.SetComponentEnabled<StrikeEvent>(entity, true)` and `entityManager.SetComponentData(entity, new StrikeEvent { NormalizedForce = 1.0f })` (fixed force per CONTEXT.md)

    **Important implementation details:**
    - Cache `EntityManager` reference in `Start()` via `World.DefaultGameObjectInjectionWorld.EntityManager`
    - Cache `Camera.main` reference in `Start()` to avoid per-frame lookup
    - Add `[SerializeField] private float maxRaycastDistance = 100f;` for configurable ray distance
    - Add a `using UnityEngine.InputSystem;` for the New Input System
    - Namespace: `SoundResonance`
    - This is NOT Burst-compiled (managed MonoBehaviour) -- that is correct and expected for the input bridge

    **Entity lookup approach:** The cleanest pattern for non-SubScene objects is to have the Baker store the entity on a companion managed component, or to maintain a static Dictionary<GameObject, Entity> populated during baking. Choose whichever approach compiles cleanly. If neither works easily, fall back to querying all entities with ResonantObjectData and matching by LocalToWorld position.

    Note: The runtime click-to-activation path is fully validated in the subsequent human checkpoint (Task 3). This task's verify focuses on compilation since the click path requires PlayMode.
  </action>

  </verify>
