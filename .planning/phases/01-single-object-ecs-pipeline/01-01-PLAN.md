  <action>
    Create the Systems/ directory if it does not exist, then create three ISystem + IJobEntity files. All three follow the same pattern from the research: `[BurstCompile] partial struct : ISystem` with a nested `[BurstCompile] partial struct : IJobEntity`.

    **EmitterActivationSystem.cs:**
    - Namespace: `SoundResonance`
    - `[UpdateInGroup(typeof(SimulationSystemGroup))]` -- no UpdateBefore/After needed since this is the first system
    - `OnUpdate`: schedule `ActivateJob` via `.ScheduleParallel()`
    - `ActivateJob.Execute` parameters: `ref ResonantObjectData data, in StrikeEvent strike, EnabledRefRW<StrikeEvent> strikeEnabled, EnabledRefRW<EmitterTag> emitterEnabled`
    - Body: `data.CurrentAmplitude += strike.NormalizedForce;` (additive energy per CONTEXT.md), `emitterEnabled.ValueRW = true;`, `strikeEnabled.ValueRW = false;` (consume event)
    - The query naturally only matches entities where StrikeEvent is enabled (IEnableableComponent default behavior)

    **ExponentialDecaySystem.cs:**
    - `[UpdateInGroup(typeof(SimulationSystemGroup))]`, `[UpdateAfter(typeof(EmitterActivationSystem))]`
    - `OnUpdate`: read `SystemAPI.Time.DeltaTime`, schedule `DecayJob` with DeltaTime field
    - `DecayJob.Execute` parameters: `ref ResonantObjectData data, in EmitterTag emitter`
    - Body: compute `decayRate = (2f * math.PI * data.NaturalFrequency) / (2f * data.QFactor)`, then `data.CurrentAmplitude *= math.exp(-DeltaTime * decayRate)`
    - Uses `in EmitterTag` (not EnabledRefRW) because it only needs to filter on enabled EmitterTag, not toggle it. The `in` parameter with an IEnableableComponent naturally filters to only enabled entities.
    - Guard: if `data.NaturalFrequency <= 0f || data.QFactor <= 0f` skip (return without modification)

    **EmitterDeactivationSystem.cs:**
    - `[UpdateInGroup(typeof(SimulationSystemGroup))]`, `[UpdateAfter(typeof(ExponentialDecaySystem))]`
    - `OnUpdate`: schedule `DeactivateJob` via `.ScheduleParallel()`
    - `DeactivateJob.Execute` parameters: `ref ResonantObjectData data, EnabledRefRW<EmitterTag> emitterEnabled`
    - Body: compare `data.CurrentAmplitude < data.DeactivationThreshold`. If below, set `data.CurrentAmplitude = 0f`, `data.Phase = 0f` (clean reset for next activation), `emitterEnabled.ValueRW = false`
    - Uses per-material `data.DeactivationThreshold` (NOT the global `ResonanceMath.AmplitudeThreshold`)

    All files must include: `using Unity.Burst;`, `using Unity.Entities;`, `using Unity.Mathematics;`, namespace `SoundResonance`.

    Do NOT use `EntityQueryOptions.IgnoreComponentEnabledState` -- the default filtering behavior is correct for all three systems.
    Do NOT use SystemBase or Entities.ForEach -- deprecated patterns.
  </action>
