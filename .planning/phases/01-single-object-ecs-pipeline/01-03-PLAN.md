  <action>
    Create PlayMode integration tests that exercise the ECS systems by creating entities programmatically and advancing the World.

    **Test infrastructure pattern:**
    ```
    [UnitySetUp] -- get World and EntityManager from World.DefaultGameObjectInjectionWorld
    [UnityTearDown] -- destroy test entities (track them in a list for cleanup)
    Helper: CreateResonantEntity(float naturalFreq, float qFactor, float deactivationThreshold) -> Entity
      - Creates entity with ResonantObjectData (with given params, CurrentAmplitude=0, Phase=0)
      - Adds EmitterTag (disabled)
      - Adds StrikeEvent (disabled)
      - Returns entity for test manipulation
    Helper: SimulateFrames(int count) -- calls World.Update() N times with yield return null between each
    ```

    **Tests to write:**

    1. **StrikeActivatesEmitter** -- Create entity, enable StrikeEvent with NormalizedForce=1.0, advance one frame, assert EmitterTag is enabled and CurrentAmplitude > 0. Assert StrikeEvent is disabled (consumed).

    2. **AmplitudeDecaysOverTime** -- Create entity with known Q-factor (e.g., Q=100, f0=200Hz for fast decay), strike it, advance several frames. Assert CurrentAmplitude is strictly less than initial post-strike value. Assert amplitude is monotonically decreasing: record amplitude after each frame and verify each value is less than the previous. Do NOT assert against an exact exponential curve -- DeltaTime in PlayMode tests is not consistent. Use monotonic decrease as the correctness check.

    3. **DeactivationAtThreshold** -- Create entity with low Q (fast decay, e.g., Q=10, f0=1000Hz) and known deactivation threshold, strike it, advance enough frames for amplitude to drop below threshold, assert EmitterTag is disabled and CurrentAmplitude == 0 and Phase == 0.

    4. **ReExcitationAddsEnergy** -- Create entity, strike it (amplitude becomes 1.0), advance a few frames (amplitude decays to some value < 1.0), record the current amplitude, strike again, advance one frame, assert amplitude > recorded value (additive, not reset to 1.0).

    5. **ReActivationAfterDeactivation** -- Create entity with fast decay, strike it, wait for deactivation, strike again, assert EmitterTag is re-enabled and CurrentAmplitude > 0.

    6. **DifferentMaterialsDifferentDecayRates** -- Create two entities: one with high Q (steel-like, Q=10000) and one with low Q (wood-like, Q=100), same frequency. Strike both, advance N frames. Assert high-Q entity has higher remaining amplitude than low-Q entity. Do NOT fit to an exact exponential curve. Just assert relative ordering: `Assert.That(steelAmplitude, Is.GreaterThan(woodAmplitude))`.

    **Important notes:**
    - Use `[UnityTest]` attribute (not `[Test]`) for PlayMode tests -- they return `IEnumerator` and can `yield return null` to advance frames
    - Use `Assert.That(value, Is.GreaterThan(expected))` style NUnit assertions
    - Do NOT use approximate exponential curve fitting assertions -- DeltaTime is inconsistent in test environments and will cause flaky tests
    - For decay tests, check monotonic decrease and relative ordering instead of exact values
    - Entity cleanup is critical -- destroy all test entities in TearDown to avoid pollution
    - The World.Update() call advances all systems (Activation, Decay, Deactivation) in correct order
    - Use `yield return null` between World.Update() calls to simulate frame boundaries
    - Namespace: `SoundResonance.Tests`
  </action>
